# Contributing to TurboQuant Technical Review

Thank you for your interest in contributing! This repository hosts an independent mathematical audit of the TurboQuant paper. All forms of contribution are welcome — agreements, rebuttals, corrections, and additional analysis alike.

---

## 📐 What Kinds of Contributions Are Welcome

- **Mathematical corrections** — if you believe any finding in this review is wrong, please show the math
- **Additional findings** — if you identify issues not covered in the current review
- **Counterarguments** — if you believe the paper's claims are actually correct, rigorous counterproofs are welcome
- **Clarity improvements** — fixing typos, improving notation, or making the exposition clearer
- **Related work** — pointers to prior art, related proofs, or follow-up papers

---

## 🚀 How to Contribute

### For small fixes (typos, formatting)
1. Fork the repository
2. Make your change directly in the GitHub editor or locally
3. Open a Pull Request with a clear description

### For substantive technical contributions
1. **Open an Issue first** — describe the claim you want to dispute or add, before writing a PR
2. Wait for brief acknowledgment (usually within a few days)
3. Fork the repo and make your changes
4. Open a Pull Request referencing the Issue

---

## 📝 Pull Request Guidelines

- Keep PRs focused on a single topic
- Use clear, descriptive titles (e.g., `Fix: lower-bound argument re sphere entropy`, `Add: discussion of Theorem 3 approximation`)
- In the PR description, clearly state: **what** you are changing and **why**
- If you are disputing a finding, please cite or derive the mathematics explicitly
- Be respectful and collegial — this is a scientific discussion, not a debate competition

---

## 🐛 Opening Issues

When opening an issue:
- Use a descriptive title
- State clearly whether you are reporting an **error in the review**, proposing an **addition**, or asking a **question**
- Include relevant math, citations, or references where applicable

---

## 📜 Licensing of Contributions

By contributing to this repository, you agree that your contributions will be licensed under the [MIT License](LICENSE).

---

## 💬 Code of Conduct

- Be respectful and professional
- Critique ideas, not people
- Back up technical claims with evidence
- Welcome diverse perspectives — the goal is to get the math right

---

*Questions? Open an issue and we will get back to you.*
